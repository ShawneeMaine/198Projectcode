Simplified Programming Rules

 

TankWarz, has a simplified programming instruction set which is designed to cover all of the major programming

functions available in a programming language. One minor difference; It will loop continuously. (i.e. it does not terminate)

This programming language is designed to allow the programmer to handle basic automated functions for a Tank in the game, allowing the user to control all aspects of the tank.

Syntax Rules

1.      Each Line in the program is treated as a step. Each step is run round-robin one at a time by the control program when the game is running.

2.      Indentation is not important.

Definitions

term

A term is a name for a set of mathematical operations that are evaluated at run time. See term rules for information on how to set up terms.

 

condition

Two terms separated by a comparator (!=, ==, <=,>=, >, or <) evaluates to true or false

 

operator

These are a part of a term. In a term any two terms may be combined by the use of an operator. Valid operators are (+, - , *, /, ^, and %(modulo)). Using an operator will create a new term. PEMDAS is followed in the case of multiple terms/operator combinations not separated by parentheses.

 

constant

A constant is a type of term that evaluates to a decimal number. It is set in the program (like 5.5)

 

variable

A variable is a named place holder for a value that can change. It is changed by means of the set command. Variables defined in a function block are only valid in that function block.

 

canned functions

A canned function is a function like command, it can either be called with the call command, or it can be treated as a term, as all canned functions return a constant.

 

KeyWords/Values:

# - At the start of a line indicates that this line is a comment(it will be ignored)

set - This word indicates that what comes after is a variable assignment. It must be followed by a variable name(any alphabetic name will do) This is followed by an equals sign and a term - (e.g. set x=4)

call - This word indicates that what comes after is a function call(either canned or user defined)

function - endfunction - This word indicates that what comes after is a function definition block. It must be terminated by a line with endfunction on it. It is immediately followed by a name for the function, and a parenthesizes set of variable names representing terms that will be passed to the fucntion when it is called. User defined functions do not return any values, all canned functions do however, so they may be used as parts of terms.

while - endwhile - This keyword combination allows the user to program a loop. It will look for a condition immediately following the while word, and if the condition evaluates to true, it will proceed to execute the following code block until it reaches the endwhile, and then begin again. It will do this until the condition is no longer true.

if - else - endif - This keyword combination allows the user to insert a switch in the program. The if is followed by a condition. If this condition is true, then the code block leading up to the endif or the optional else is executed. If not then either the code in an optional else block or immediately following the endif (required) will be executed.

break/continue - The break and continue commands will only work within a while loop. The break will exit the loop immediately, while the continue will send control back to the beginning of the loop.

return - The return command will exit a function and return control to the point where it left the calling program.

 

Canned Functions

 

All angles are in degrees

abs(s)- This function takes a value and return its absolute value.

acos(s)- This function takes a value and return its arccosine.

ammoset(id)- This function will allow you to switch between ammo types. It takes the index of the ammo(0-n) and returns the index set. It will default to 0 in the case of an invalid id.

asin(s)- This function takes a value and return its arcsine.

atan(s)- This function takes a value and return its arctangent.

dist(x,y,x1,y1)- This function will calculate and return the distance between two points on the map.

cos(angle)- This function takes an angle(degrees) and return its cosine.

fire(x,y)- This function will fire the current ammo at the specified position, given that the position is within range, and the ammo is loaded. It returns 0 on success 1 on failure.

firetarget(id)- This function will fire the current ammo, if it has homing capability, at the specified target so long as that target is within range of the weapon, and is currently in scanner range. It returns 0 on success 1 on failure. The missile will follow its target until it hits it.

firetrack(id)- This function will fire the current ammo at the specified target so long as that target is within range of the weapon, and is currently in scanner range. It returns 0 on success 1 on failure.

followtarget(id)- This function will set your tank's current destination to the position of the target id, so long as the target id is in scanner range. It returns the id of the target on success, and your id on failure.

getarmordamage(id)- This function takes a single argument of a tank id which is within your scanner area.  It will return a number between 0 and 100.  This number indicates a percentage of how much damage the tank’s armor can still sustain.  It is a percentage of armor left.  If the id specified is not within scanner range, the function will return  -1.

getarmorstrength(id)- This function takes a single argument of a tank id which is within your scanner area.  It will return the total amount of armor strength remaining on the tank.  If the id specified is not within scanner range, the function will return  -1.

getchassisdamage(id)- This function takes a single argument of a tank id which is within your scanner area.  It will return a number between 0 and 100.  This number indicates a percentage of how much damage the tank’s chassis can still sustain.  It is a percentage of overall chassis strength left.  If the id specified is not within scanner range, the function will return  -1.

getchassisstrength(id)- This function takes a single argument of a tank id which is within your scanner area.  It will return the total amount of chassis strength remaining on the tank.  If the id specified is not within scanner range, the function will return  -1.

getgunrange()- This function takes no arguments, and will return the maximum range of your gun.

getid()- This function takes no arguments, and will return the id of your tank.

getscanarc()- This function takes no arguments, and will return the arc of your tank scanner.

getxpos(id)- This function will return the x position of the given target. If the target is not within scanner range it will return the x position of your tank.

getypos(id)- This function will return the y position of the given target. If the target is not within scanner range it will return the y position of your tank.

godest(x,y)- This function will set the destination for your tank. If the destination is off of the board, it will return a 1, if you are within a tank width of the destination, it will return a 2 otherwise it will set you destination and return a 0.

iamscanned()- This function will return a 0 if your tank is currently within another tank’s scanner area.  It will return a 1 otherwise.

inrange(id)- This function will tell you if a tank which is currently scannable is within your weapons range. It returns a 1 if it is and a 0 if it isn't.

maxx()- This function takes no arguments, and will return the maximum width of the field.

maxy()- This function takes no arguments, and will return the maximum height of the field.

minedrop()- This function will set a mine on the field at a distance of (30,30) + the proximity of the currently active mine. There is no return value for this function.

mineset(id)- This function will allow you to switch between mine types. It takes the index of the mine(0-n) and returns the index set. It will default to 0 in the case of an invalid id.

minesetproximity(value)- This function will allow you to set the range at which a mine will explode. If you specify a range that is beyond the maximum range for the currently selected mine, then the maximum range will be used. If a proximity is specified that is less than 0, then 0 will be used. This function returns the range set on the mine.

minesettarget(id)- This function will allow you to set a specific target that will set off the mine when laid. If the target is not within scanner range, the it returns a 0. If the target is invalid or the mine does not support targeting, then it will return a -1, else it will return a 1.

print(d)- This function will print a term to the output area on the main programming screen. It is a diagnostic tool when you are programming.

random()- This function will return a random number from 0 to 100.

scanclosest()- This function will scan for the closest object and return its id. If there is no opposing tank scanned, then the id of your tank will be returned. You can use the getid() function to get your tank's id number.

scanresetarc()- This function will change the arc of the scanner to the default value and the range to the default value for the scanner - returns the arc.

scansetangle(angle)- This function will change the angle of the scanner to the angle specified - returns the angle.

scansetarc(angle)- This function will change the arc of the scanner to the angle specified. It will also modify the range to the maximum value possible for the arc specified. - returns the range.

scansetrange(distance)- This function will change the range of the scanner to the distance value specified. It will also set the arc to the maximum arc allowed for the range specified. If the arc exceeds 359 degrees it will be set to 359 degrees. - returns the arc.

scansetdest(x,y)- This function will angle the scanner to point in the direction of a location on the field - returns the angle.

sin(angle)- This function takes an angle(degrees) and return its sine.

sqrt(v)- This function will return the square root of v.

tan(angle)- This function takes an angle(degrees) and return its tangent. It will allow angles like 90 or 270, but will only approximate them.
